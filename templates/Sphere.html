SPHERE ERP

Simple overview of how a person uses the system from start to finish.

What this system does

  • Create a request to buy items or services.
  • Send the request for approval based on amount and role.
  • Turn an approved request into a purchase order.
  • Check budget, pick a vendor, and auto fill details.
  • Generate the document and notify the right people.
  • Keep a clear record of every step.

End-to-End user journey

flowchart TD A[Open login page] --> B{Have an account} B -- Yes --> C[Enter email and password] B -- No --> S[Sign up with basic details] S --> C C --> D[Login successful] D -- No --> A D -- Yes --> E[See dashboard] E --> E1[See tasks and status] E --> E2[View profile] E --> F[Create purchase request] F --> G[Fill request details] G --> H[Attach files if needed] H --> I[Submit for approval] I --> J[Approver decision] J -- Reject --> J1[Share reason] J1 --> J2[Edit request] J2 --> I[Submit for approval] J -- Approve --> K[Request is approved] K --> L[Generate purchase order] L --> M[Pick vendor and items] M --> N[System checks budget] N --> O{Budget ok} O -- No --> O1[Create NFA note] O1 --> P[Send for higher approval] O -- Yes --> P[Send for approval] P --> Q{Amount level} Q -- Low --> Q1[Manager approves] Q -- Medium --> Q2[Head approves] Q -- High --> Q3[CFO or CEO approves] Q1 --> R[Final approval] Q2 --> R Q3 --> R R --> T[Make PO document] T --> U[Send email to people] U --> U1[People see alerts] U --> V[PO ready for action] V --> W[Keep records and history] E --> X[Log out] W --> X[Log out]

Notes: The path changes by amount and role. When budget is short, the system asks for an extra note and higher approval.
